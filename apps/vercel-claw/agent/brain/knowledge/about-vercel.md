# Vercel

As of April 2026, Vercel is a managed cloud platform for deploying and operating web applications. In technical terms, it takes application source code, runs a build pipeline, produces deployment artifacts, serves static assets from a global CDN, and executes dynamic server-side code in Vercel Functions. It also provides preview deployments, HTTPS, routing, environment management, observability, and deployment controls. Vercel is most closely associated with Next.js, but it also supports other frontend and backend frameworks.

Its compute model is fundamentally serverless. That does not mean there are no servers; it means the developer does not provision or manage them directly. Instead, Vercel allocates compute on demand when requests arrive, scales capacity automatically, and manages instance lifecycle, regional placement, and failover behavior. On current Vercel projects, this commonly uses Functions with Fluid Compute, which keeps the on-demand properties of serverless execution while improving utilization by allowing multiple invocations to share an instance when capacity is available. In short, Vercel sells managed builds, global content delivery, and on-demand application compute.
